[로그인 개념]
4강 강의자료: 쿠키와 세션

쿠키 - 현재 내가 사용하는 브라우저에 저장되는 사용자 정보 (브라우저마다 따로 적용)
-> 로그인 정보를 쿠키에 저장하면 위험함!!

세션 - 사용자 정보를 서버(WAS)에서 관리

Request/Response : JSessionId으로

(사진참고)
1. 브라우저 -> WAS
   003(쿠키)    003(세션)
2. WAS -> 브라우저
   003(세션)    003(쿠키)

세션 정보가 사라지는 경우
1. 특정시간 경과
2. 로그아웃 or 브라우저 닫기
브라우저 -> WAS
003(쿠키)    X(세션)
=> 이제 요청해도 해당 JSessionId 세션이 없음

* 세션에는 필요한 정보(유저에 관한 간략한 정보)만 담기
-> 기본적으로는 DB에 접근해서 정보가져와야함

1. 세션 생성 -> 정확히는 지금 사용자 브라우저의 세션을 "가져오는" 것
방법1) HttpSession session
방법2) HttpServletRequest request
HttpSession session = request.getSession();

2. HTML에서 ${sessoin.userId} 형식으로 세션에 접근
